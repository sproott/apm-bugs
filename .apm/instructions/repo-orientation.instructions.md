# apm-bugs — Repo Orientation

This repo is a collection of **minimal reproductions of bugs in the APM (Agent
Package Manager) CLI**, each paired with a filed-issue write-up. It is itself an
APM package (`apm.yml`, target `claude`) but ships to no consumers — it exists to
hold reproductions.

## APM source of truth

The APM CLI source is vendored as a git submodule at **`@source/apm`**
(`git@github.com:microsoft/apm`). Read it directly — never guess CLI behavior or
internals from memory:

- Python package: `@source/apm/src/apm_cli/`
- Commands: `@source/apm/src/apm_cli/commands/` (e.g. `prune.py`, `deps/cli.py`,
  `deps/_utils.py`, `_helpers.py`)
- Policy / CI checks: `@source/apm/src/apm_cli/policy/ci_checks.py`
- Models: `@source/apm/src/apm_cli/models/apm_package.py`

When citing source in an issue, link to the canonical GitHub blob, e.g.
`https://github.com/microsoft/apm/blob/main/src/apm_cli/commands/prune.py` — the
path after `src/` matches the submodule layout.

Always record the `apm --version` you actually observed the bug on (the installed
CLI may be a different version than the submodule checkout — verify both).

## File structure

- `README.md` — index of all bugs + a per-bug section (setup + `make` reproduce).
- `issue-<slug>.md` — one filed-issue write-up per bug (GitHub bug template).
- `handoff-<slug>.md` — an investigation lead handed off for a future session;
  keeps its investigation framing (it is not a filed issue).
- `<bug-slug>/` — one directory per bug holding the runnable reproduction: a
  checked-in `apm.yml` (+ nested `package/` etc.) and a `Makefile` with reproduce
  targets. `apm_modules/` and `apm.lock.yaml` are git-ignored (regenerated by
  `apm install`).
- `.apm/` — this package's authored primitives (instructions, skills). Edit here,
  then `apm install`; never edit deployed output under `.claude/`.

**Git-ignore ownership**: a `.gitignore` pattern that only makes sense for one
reproduction directory belongs in that directory's own `.gitignore`, not the root
one. Keep the root `.gitignore` for repo-wide patterns (`apm_modules/`,
`apm.lock.yaml`, deployed target dirs); scope demo-specific ephemera (e.g. an
isolated `fake-home/`) to the subdir it belongs to.

This repo is APM-managed: `AGENTS.md` / `CLAUDE.md` / `.claude/rules/*` (if
present) are compiled from `.apm/` and are clobbered on `apm install` — author
global instructions in `.apm/instructions/<name>.instructions.md`, not in the
compiled outputs.
